DEVICE_DT_DEFINE 一行宏 · 三件事
dts 节点 + initcall + ops 表 · 缝合在一起
一行宏 驱动注册的入口 
DEVICE_DT_DEFINE(
    DT_NODELABEL(leds),                // (1) dts
    led_gpio_init,                       // (2) initcall
    NULL,
    &led_gpio_data,
    &led_gpio_cfg,
    POST_KERNEL,
    CONFIG_LED_INIT_PRIORITY,
    &led_gpio_api                          // (3) ops
);
1 绑定 dts 节点
生成 static const struct device __device_dts_ord_<N> 静态对象 · 编译期就有 · 一个节点对应一个对象
2 挂 initcall
把 led_gpio_init 挂到 linker section .z_init_<level><prio>_* · 启动时按级别顺序自动跑
3 绑 ops 表
device 对象内部 api 字段指向 &led_gpio_api · 上层调虚函数自动派发到这里
dts + initcall + ops · 三件套缝合 · 字面工程化
