# 安全相关

## 报告安全问题

这个仓库是教学用的，本身不直接连网络、不处理用户数据。但如果你发现：

- 配套代码里有缓冲区溢出 / 未定义行为 / 内存泄漏 / 整数溢出
- 文档里讲的某种写法本身存在安全隐患（比如错误地教了 `strcpy`）
- 工业实战章节里残留了不该公开的真实数据或真实凭证

**不要在公开 Issue 里直接讨论**，先发邮件到我个人邮箱，给我修复时间。

邮箱：通过公众号「兆鸣嵌入式」回复"安全"获取。

收到后我一般 48 小时内回。修复 commit 里我会致谢。

## 不算安全问题的

普通的 typo、概念错讲、`#define` 写错、章节顺序排错——这些直接开 Issue，越多越好。

不影响代码安全的代码风格争议（命名、缩进、注释多寡）——直接开 Issue 或 PR 讨论。
